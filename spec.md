
# 사내 회의실 예약 웹앱 — MVP Spec

> 사내 AI 해커톤 출품 / 데모용 프로젝트
> 작성일: 2026-06-24 | 기간: 3주 | 팀: 3명

---

## 1. 한 줄 정의

사내 회의실의 **빈 시간을 실시간으로 확인**하고 **중복 없이 예약**할 수 있는 웹앱.
엑셀·메신저로 흩어진 회의실 예약을 한곳으로 모은다.

---

## 2. 문제

**현재 상황**

- 회의실 예약을 **엑셀 시트 / 메신저 / 구두**로 제각각 관리 중
- 어느 회의실이 비어 있는지 한 번에 파악하기 어렵다
- 같은 시간에 같은 회의실을 두 명이 잡는 **중복 예약**이 종종 발생한다
- 예약만 해두고 쓰지 않는 **노쇼** 때문에 빈 회의실이 낭비된다
- 총무 담당자가 전체 회의실 사용 현황을 모아서 볼 방법이 없다

**해결하려는 것**

- 회의실 현황을 한 화면에서 실시간으로 보여주고
- 예약 시점에 중복을 막아 주며
- 누가 언제 어디를 예약했는지 기록으로 남긴다

---

## 3. 사용자 (페르소나)

| 사용자                  | 역할         | 주요 작업                                       |
| ----------------------- | ------------ | ----------------------------------------------- |
| **직원 (예약자)** | 1순위 사용자 | 회의 잡을 때 빈 회의실을 빠르게 찾아 예약·취소 |
| **총무/관리자**   | 운영자       | 회의실 등록·수정, 전체 예약 현황 조회          |

- **직원**: "수요일 오후 2시에 4명 들어갈 회의실 어디 비어 있지?"를 30초 안에 해결하고 싶다.
- **관리자**: 회의실 정보를 직접 관리하고, 어떤 방이 많이 쓰이는지 보고 싶다.

---

## 4. 핵심 기능 (MVP — 3주)

### 4.1 회의실 목록 & 실시간 현황

- 전체 회의실을 카드/목록으로 표시
- 각 회의실의 이름, 위치(층), 수용 인원, 보유 장비(프로젝터/화상 등) 표시
- 현재 시각 기준 **사용 중 / 비어 있음** 상태 배지

### 4.2 예약 생성 (중복 방지)

- 날짜 → 회의실 → 시간대(시작/종료) 선택 후 예약
- 회의 제목, 예약자, 참석 인원 입력
- **같은 회의실·겹치는 시간대 예약은 차단** (충돌 검사)
- 예약 완료 화면 + 예약 내역 저장

### 4.3 캘린더 / 타임라인 뷰

- 일(Day) / 주(Week) 단위로 회의실별 예약 현황을 타임라인으로 표시
- 빈 시간대를 한눈에 확인 가능
- 타임라인의 빈 칸 클릭 시 바로 예약 폼으로 연결

### 4.4 내 예약 관리

- 내가 만든 예약 목록 조회
- 예약 수정 / 취소
- 다가오는 예약 표시

### 4.5 관리자 — 회의실 관리

- 회의실 추가 / 수정 / 삭제 (CRUD)
- 회의실 정보(이름, 위치, 수용 인원, 장비) 관리
- 전체 예약 현황 조회

### 4.6 예약 알림 (이메일)

- 예약 생성 시 예약자에게 확인 메일 발송
- 회의 시작 전 리마인드 메일 발송

### 4.7 체크인 & 노쇼 자동 취소

- 회의 시작 시각 이후 일정 시간(예: 10분) 내 **체크인** 버튼 클릭
- 체크인이 없으면 예약 자동 취소 → 회의실 다시 개방

---

## 5. 제외 범위 (이번 MVP에서 안 함)

- ❌ 외부 캘린더 연동 (Google Calendar / Outlook)
- ❌ 화상회의 솔루션 연동 (Zoom / Teams 자동 링크 생성)
- ❌ 모바일 네이티브 앱 (웹 반응형만)
- ❌ 비품·다과·케이터링 예약
- ❌ 다중 지점/건물 (단일 사옥만)
- ❌ 결제·유료 회의실
- ❌ 반복 예약 (매주 정기 회의 자동 등록)

---

## 6. 성공 기준 (시연 시나리오)

다음을 화면에서 보여줄 수 있으면 **성공**:

1. ✅ 회의실 목록에서 비어 있는 방과 사용 중인 방이 상태로 구분되어 표시됨
2. ✅ 날짜·회의실·시간을 선택해 예약을 생성하고, 목록/캘린더에 즉시 반영됨
3. ✅ 이미 예약된 시간대와 겹치는 예약을 시도하면 **차단**됨
4. ✅ 주간 타임라인에서 빈 시간대가 한눈에 보임
5. ✅ 내 예약 목록에서 예약을 취소하면 해당 시간대가 다시 열림
6. ✅ 관리자가 회의실을 추가/수정하면 예약 화면에 반영됨
7. ✅ 예약 생성 시 확인 이메일이 발송됨
8. ✅ 체크인하지 않은 예약이 자동 취소되어 회의실이 다시 개방됨

## 8. 데이터 모델 (초안)

```sql
-- 사용자
users (
  id            uuid PRIMARY KEY,
  email         varchar UNIQUE NOT NULL,
  name          varchar NOT NULL,
  role          varchar NOT NULL,   -- employee / admin
  created_at    timestamp
)

-- 회의실
rooms (
  id            uuid PRIMARY KEY,
  name          varchar NOT NULL,   -- 예: "대회의실 A"
  floor         varchar,            -- 위치(층)
  capacity      integer,            -- 수용 인원
  equipment     varchar[],          -- 보유 장비(프로젝터, 화상 등)
  is_active     boolean DEFAULT true,
  created_at    timestamp
)

-- 예약
reservations (
  id            uuid PRIMARY KEY,
  room_id       uuid REFERENCES rooms(id),
  user_id       uuid REFERENCES users(id),
  title         varchar NOT NULL,   -- 회의 제목
  attendees     integer,            -- 참석 인원
  start_at      timestamp NOT NULL, -- 시작 일시
  end_at        timestamp NOT NULL, -- 종료 일시
  status        varchar NOT NULL,   -- booked / checked_in / cancelled / no_show
  checked_in_at timestamp,
  created_at    timestamp
)
```

> 중복 예약 방지: `room_id` + 시간대(`start_at` ~ `end_at`) 겹침 검사 후 INSERT.

## 9. 개발 일정 (3주, 3명)

| 주차          | 마일스톤   | 작업                                                                      |
| ------------- | ---------- | ------------------------------------------------------------------------- |
| **1주** | 기반 구축  | DB 스키마·Supabase 설정, 로그인, 회의실 목록 UI                          |
| **2주** | 예약 핵심  | 예약 생성(중복 방지), 캘린더/타임라인 뷰, 내 예약 관리                    |
| **3주** | 부가·완성 | 관리자 회의실 관리, 이메일 알림, 체크인/노쇼 취소, 통합 테스트·시연 준비 |

**팀 역할 분담 (추천)**

- 1명: 백엔드 + DB + 예약 충돌 검사 로직
- 1명: 프론트엔드(목록·예약 폼·캘린더 뷰)
- 1명: 관리자 화면 + 이메일 알림 + 체크인/노쇼
