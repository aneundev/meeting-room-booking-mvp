### 핵심 강점

1. ✅ **문제가 구체적이고 공감됩니다**
   - 「회의실 예약을 엑셀 / 메신저 / 구두로 제각각 관리」
   - 「같은 시간에 같은 회의실 중복 예약이 종종 발생」
   - 「예약만 해두고 안 쓰는 노쇼 낭비」

2. ✅ **사용자 페르소나가 명확합니다**

   - 관리자: 예약 상황 전체 파악 및 노쇼 관리

3. ✅ **데이터 모델이 견고합니다**
   - users / rooms / reservations 3개 테이블 정의

5. ✅ **제외 범위가 명확합니다**

## 🔴 핵심 리스크

### 1. 시간 부족 (가장 중요)
- 이전: 5페이지 CRUD → 1~2주
- 현재: 8~10페이지 + 복잡한 로직 → 30시간은 **타이트**
- 특히 **캘린더 UI + 중복 검사 로직**이 동시에 필요한데 시간 부족

### 2. 기능 7개가 확산되어 있음
**우선순위 (명확히)**:
- **핵심** (꼭 필요): 회의실 목록 + 예약 생성 + 내 예약 관리 + 중복 검사
- **부가** (시간 남으면): 캘린더 뷰
- **옵션** (v2로 미루기): 이메일 알림, 체크인/노쇼 자동 취소

### 3. AI 활용이 현재 없음
- **Claude 해커톤인데 AI가 없다는 게 가장 큰 약점**
- 표준 CRUD만으로는 경쟁력 부족
- 추천: **자연어 예약 기능** 한 줄기만 추가해도 차별점 확보

### 4. 중복 방지 로직은 신뢰의 전부
- 이 앱의 모든 신뢰는 「정말 겹치는 예약이 막히는가」에 달림
- 경계 케이스 확인 필수:
  - 끝나는 시각과 시작 시각이 정확히 맞닿는 경우
  - 동시 다중 요청 (동시성 제어)

## 🟢 다음 단계

### 옵션 A: 범위 재조정 (✅ 권장)

**MVP (3주 완성)**
- 회의실 목록 조회
- 예약 생성 (중복 검사 포함)
- 내 예약 관리 및 취소
- **데모 척추**: 「빈 회의실 확인 → 예약(중복 차단) → 취소」

**V2 (이후 확장)**
- 캘린더 타임라인 뷰
- 이메일 알림
- 체크인/노쇼 자동 취소
- **자연어 예약** (AI 활용)

### 옵션 B: 주차별 마일스톤 (구체적 계획)

**1주차**: DB 설계 + 로그인 + 회의실 목록 조회
**2주차**: 예약 생성 + 중복 검사 로직 + 내 예약 관리
**3주차**: 통합 테스트 + 샘플 데이터 준비 + 시연 리허설

### 옵션 C: AI 기능 추가 (차별화)

**자연어 예약** (Claude API 활용):
- 입력: 「내일 오후에 6명 회의실 잡아줘」
- 처리: Claude가 자연어 파싱 → 빈 방 검색 → 자동 예약
- 효과: AI 해커톤 적합성 ↑ + 차별점 확보

**다른 아이디어**:
- 「참석자 수/회의 목적에 맞는 회의실 추천」
- 「자동 일정 최적화」
